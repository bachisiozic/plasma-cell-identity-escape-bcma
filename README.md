# Plasma cell identity escape drives resistance to anti-BCMA T-cell–redirecting therapy in multiple myeloma

This repository contains the analysis code accompanying the manuscript:

> **Plasma cell identity escape drives resistance to anti-BCMA T-cell–redirecting therapy in multiple myeloma.**
> *Ziccheddu B., Maura F., et al.* `<journal>`, `<year>`. DOI: `<doi>`.

The study integrates two complementary data modalities to understand why some multiple
myeloma patients lose durable response to BCMA-targeted T-cell–redirecting therapy
(CAR-T and bispecific antibodies, "TCE"):

1. **Whole-genome sequencing (WGS) genomics** — copy-number, structural variants,
   non-synonymous mutations and driver genes; survival modelling; biallelic *TNFRSF17*
   (BCMA) loss; and a combined "complex genomics + plasma-cell-identity loss" signature.
2. **Single-cell RNA-seq (scRNA-seq)** — the malignant plasma-cell compartment and the
   surrounding immune microenvironment, comparing durable (DR) versus non-durable (NDR)
   responders.

Throughout, **DR = durable response** (progression-free survival > 180 days) and
**NDR = non-durable response**.

> **Original filenames.** The scripts were renamed for clarity. The mapping is:
> `01_genomic_data_preparation.R` ← `create_final_files.R`,
> `02_genomic_analysis.R` ← `final_github_script.R`,
> `03_scRNAseq_analysis.Rmd` ← `scRNAseq_analysis_report_29JUN26.Rmd`.

The two modalities are **independent**: you can run the genomic analysis without the
single-cell analysis and vice versa.

## Software and dependencies

Analyses were run in **R (≥ 4.2)**. Install the packages below before running.

**Genomic analysis (`01`–`02`)**

```r
install.packages(c("stringr","dplyr","plyr","tidyr","tidyverse","data.table",
                   "ggplot2","survival","survminer","RColorBrewer","colorspace",
                   "reshape","readxl","pheatmap","forestmodel","deconstructSigs",
                   "quantsmooth"))
# Bioconductor
if (!requireNamespace("BiocManager", quietly = TRUE)) install.packages("BiocManager")
BiocManager::install(c("GenomicRanges","IRanges","GenomicFeatures","maftools",
                       "ComplexHeatmap","biomaRt"))
# specialised (myeloma) packages
# mmsig:   https://github.com/UM-Myeloma-Genomics/mmSig
# dndscv:  https://github.com/im3sanger/dndscv
```

**Single-cell analysis (`03`)**

```r
install.packages(c("Seurat","SeuratObject","harmony","dplyr","stringr","tidyr","ggplot2"))
if (!requireNamespace("BiocManager", quietly = TRUE)) install.packages("BiocManager")
BiocManager::install(c("DESeq2","edgeR","fgsea","msigdbr"))
```

## How to reproduce

### Genomic analysis

1. Place the input tables listed in [`data/README.md`](data/README.md) into `data/`.
2. *(Optional, only if rebuilding the derived tables from raw cohort files)* run
   `scripts/01_genomic_data_preparation.R`. Most users can skip this and use the provided
   `ALL_*_FINAL.txt` tables directly.
3. Run `scripts/02_genomic_analysis.R`. This reproduces the survival models, driver
   screens, the biallelic-BCMA analysis, the bulk-RNA correlations, and the genomic
   figures.

### Single-cell analysis

The report is written so it **displays the code without re-executing it** (`eval = FALSE`
in the setup chunk), so it knits on any machine without the large Seurat objects:

```r
rmarkdown::render("scripts/03_scRNAseq_analysis.Rmd")
```

To actually recompute results, obtain the Seurat objects (see `data/README.md`), update
the paths at the top of each chunk, and set `eval = TRUE`.

## Data availability

Patient-level sequencing data are subject to controlled access and are **not** stored in
this repository. This repo provides:

- the **analysis code** (`scripts/`),
- the **gene sets** needed for enrichment (`gene_sets/`),
- small, de-identified **derived tables** that reproduce the final figures (`results/`).

Large objects (Seurat `.RDS`, WGS calls, raw RNA-seq) should be deposited in an
appropriate repository (e.g. EGA / dbGaP for controlled-access sequencing, GEO for
expression, Zenodo for processed objects) and linked here:

- Processed Seurat objects: `<Zenodo/GEO accession>`
- WGS / variant calls: `<EGA/dbGaP accession>`

See [`data/README.md`](data/README.md) for the exact input files each script expects.
